State provides tools for creating resources in F5 Distributed Cloud.

## Tools

| Tool | Description |
|------|-------------|
| `f5xc-api-users-state-create` | Set Token State. |

## Parameters

### Path Parameters

| Parameter | Description | Example |
|-----------|-------------|--------|
| `name` | Token name Token object name to change, it's usually same as metadata.uid. | `my-resource` |
| `namespace` | Token namespace Namespace where token is residing. | `system` |


## Example Usage

Ask Claude to help you work with State resources:

### Create State

> "Create a state named 'example' in the 'production' namespace"

## CURL Examples

```bash
# Create resource
curl -X POST "https://${TENANT}.console.ves.volterra.io/api/register/namespaces/$${PARAM}/tokens/$${PARAM}/state" \
  -H "Authorization: APIToken ${F5XC_API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d @state.json

```
