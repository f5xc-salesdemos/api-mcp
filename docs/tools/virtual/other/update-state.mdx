Update State provides tools for creating resources in F5 Distributed Cloud.

## Tools

| Tool | Description |
|------|-------------|
| `f5xc-api-virtual-update-state-create` | Update Vulnerabilities for Virtual Host. |

## Parameters

### Path Parameters

| Parameter | Description | Example |
|-----------|-------------|--------|
| `name` | Virtual Host Name Virtual Host name for current request. | `example-resource` |
| `namespace` | Namespace Namespace of the virtual host for current request. | `system` |


## Configuration Choices

This resource includes mutually exclusive configuration options:

### context

| Option | Description | Recommended |
|--------|-------------|-------------|
| `api_endpoint` | Api endpoint |  |


## Example Usage

Ask Claude to help you work with Update State resources:

### Create Update State

> "Create a update-state named 'example' in the 'production' namespace"

## CURL Examples

```bash
# Create resource
curl -X POST "https://${TENANT}.console.ves.volterra.io/api/ml/data/namespaces/$${PARAM}/virtual_hosts/$${PARAM}/vulnerability/update_state" \
  -H "Authorization: APIToken ${F5XC_API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d @update_state.json

```
