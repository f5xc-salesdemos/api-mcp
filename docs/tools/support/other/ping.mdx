Ping provides tools for creating resources in F5 Distributed Cloud.

## Tools

| Tool | Description |
|------|-------------|
| `f5xc-api-support-ping-create` | Ping. |

## Parameters

### Path Parameters

| Parameter | Description | Example |
|-----------|-------------|--------|
| `namespace` | Namespace Namespace for which the request is sent (system) | `system` |
| `site` | Site Name Name of the site from which ping is initiated. | `-` |


## Configuration Choices

This resource includes mutually exclusive configuration options:

### hostname_or_ip

| Option | Description | Recommended |
|--------|-------------|-------------|
| `dest.hostname` | Hostname |  |
| `dest.ip` | Ip |  |

### interface_choice

| Option | Description | Recommended |
|--------|-------------|-------------|
| `intf.any_intf` | Any intf |  |
| `intf.intf` | Intf |  |


## Example Usage

Ask Claude to help you work with Ping resources:

### Create Ping

> "Create a ping named 'example' in the 'production' namespace"

## CURL Examples

```bash
# Create resource
curl -X POST "https://${TENANT}.console.ves.volterra.io/api/operate/namespaces/$${PARAM}/sites/${PARAM}/ver/ping" \
  -H "Authorization: APIToken ${F5XC_API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d @ping.json

```
