Pdf provides tools for creating, and listing resources in F5 Distributed Cloud.

## Tools

| Tool | Description |
|------|-------------|
| `f5xc-api-servicemesh-pdf-create` | Get Service API Endpoint PDF. |
| `f5xc-api-servicemesh-pdf-list` | Get PDF. |

## Parameters

### Path Parameters

| Parameter | Description | Example |
|-----------|-------------|--------|
| `app_type_name` | App Type App Type for current request. | `-` |
| `namespace` | Namespace Namespace of the App type for current request. | `system` |
| `service_name` | Service Identifies the destination service. | `-` |

### Query Parameters

| Parameter | Description | Example |
|-----------|-------------|--------|
| `collapsed_url` | API endpoint for which PDFs are requested. | `-` |
| `method` | Method of API endpoint for which PDFs are requested. | `-` |


## Example Usage

Ask Claude to help you work with Pdf resources:

### Create Pdf

> "Create a pdf named 'example' in the 'production' namespace"

### List Pdfs

> "List all pdfs in the 'production' namespace"

## CURL Examples

```bash
# List resources
curl -X GET "https://${TENANT}.console.ves.volterra.io/api/ml/data/namespaces/$${PARAM}/app_types/${PARAM}/api_endpoint/pdf" \
  -H "Authorization: APIToken ${F5XC_API_TOKEN}"

# Create resource
curl -X POST "https://${TENANT}.console.ves.volterra.io/api/ml/data/namespaces/$${PARAM}/app_types/${PARAM}/services/${PARAM}/api_endpoint/pdf" \
  -H "Authorization: APIToken ${F5XC_API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d @pdf.json

```
